# Docs

Research, plan and design for the unified AI work-feed app.

| Doc | What it covers |
|---|---|
| [product-brief.md](product-brief.md) | The vision: problem, target user, core loop, scope |
| [product-research.md](product-research.md) | Cited market research: competitive landscape, white space, AI prioritization, monetization |
| [capabilities.md](capabilities.md) | **What we can actually build.** Read-only sweep of every platform (GitHub, Slack, Calendar, Drive, Docs, Gmail, Linear, Notion, Jira): what we can show, what the user can act on, what fires in real time, plus the unified notification map and shortlist |
| [mvp-plan.md](mvp-plan.md) | **The build plan.** Stack, access checklist, database schema, clusters, notification and anxiety strategy, AI role, phases, and how to run it on Mac, Android and iOS |
| [design-system.md](design-system.md) | Locked design tokens for the chosen direction |
| [mockups/explorations.html](mockups/explorations.html) | **25 screens.** Five premium directions (palette + layout) across all five screens. Pick one |
| [mockups/screens.html](mockups/screens.html) | The original single-direction mockup (superseded by explorations) |

**Read order for review:** `mockups/explorations.html` first (pick a direction), then `capabilities.md` (shortlist what to show), then `mvp-plan.md` (how it gets built).

**On evidence:** `capabilities.md` was produced by read-only discovery with **no live calls against connected accounts**. `product-research.md` separates **Verified** claims from **Supporting** and **Refuted/Open**; do not build on anything outside the Verified tier without re-confirming.
